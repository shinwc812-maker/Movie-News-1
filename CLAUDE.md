# Claude/AI Collaboration Rules

이 프로젝트에서 AI 어시스턴트(Claude 등)는 다음 규칙을 엄격히 준수해야 합니다.

### 1. 효율성 및 토큰 최적화 (Efficiency & Token Usage)
- **최소 변경 원칙 (Surgical Updates):** 코드 수정 시 전체 파일을 다시 쓰지 않습니다. 변경이 필요한 부분만 정확히 제공하며, 변경되지 않은 부분은 `// ... existing code ...`와 같이 생략합니다.
- **단계별 사고 (Chain of Thought):** 복잡한 작업은 답변 전 내부적으로 논리를 설계한 뒤 진행합니다.
- **불필요한 서론 생략:** 인사말이나 반복적인 확인 멘트보다는 기술적인 해결책과 코드 위주로 답변합니다.

### 2. 기술적 정밀도 (Technical Precision)
- **임의 추측 금지 (No Guessing):** 요청이 모호하거나 정보가 부족한 경우, 추측하여 구현하지 말고 반드시 사용자에게 질문하여 명확히 합니다.
- **일관성 유지 (Consistency):** 프로젝트의 기존 코드 스타일, 명명 규칙(Naming Convention), 아키텍처 패턴을 분석하고 이를 엄격히 따릅니다.
- **무분별한 탐색 금지 (Targeted Exploration):** 작업과 직접 관련이 없는 파일을 무분별하게 읽거나 검색하지 않습니다. 필요한 범위 내에서만 효율적으로 탐색합니다.

### 3. 커뮤니케이션 태도 (Communication & Professionalism)
- **전문적 어조:** 반말이나 상스러운 표현을 절대 사용하지 않으며, 정중하고 전문적인 어조를 유지합니다.
- **명확한 근거:** 제안이나 수정 사항에 대해 기술적인 이유와 의도를 간결하게 설명합니다.

---
*이 규칙은 프로젝트의 안정성을 유지하고 AI와의 협업 효율을 극대화하기 위해 작성되었습니다.*
